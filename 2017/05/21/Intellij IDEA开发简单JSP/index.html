Intellij IDEA开发简单JSP

一.安装Intellij IDEA

1.下载

2.汉化 将下载汉化包(新版本汉化包是.rar格式，旧版是.jar格式)放在根目录的lib文件夹下，重启程序。

3.license server： http://jetbrains.tech

二.安装Java jdk

1.下载

2.配置环境变量

  • 系统变量→新建 JAVA_HOME 变量 变量值填写jdk的安装目录
  • Path变量→编辑在变量值最后输入 %JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;
  • 系统变量→新建 CLASSPATH 变量变量值填写 .;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar（注意最前面有一点）

3.检验是否配置成功。
cmd

出现版本信息则成功。

jdk和jre是一起安装的不需要单独下载安装jre。

三.安装Tomcat

  1. .下载

  2. 配置环境变量
    新建CATALINA_HOME 例如：C:\Program Files\Apache Software Foundation\Tomcat 8.0\bin

  3. 测试是否成功

如果没有加入服务,进入bin目录执行service.bat文件,然后使用startup.bat启动服务，shutdown.bat停止服务。最后http://localhost:8080/看到欢迎界面说明成功。

  7. 在 Artifacts > Output Layout 中，将右侧 Available Elements 中的 lib 文件夹加入到左侧，使得 lib 中的文件会在编译的时候输出：

  8. Tomcat 目录下有一个 webapps 文件夹，是服务启动的根目录，类似于 Apache 的 htdocs。当启动 Tomcat 服务时，访问 localhost:8080 相当于访问 webapps 目录；

    如果我们把项目输出的文件放到 webapps 中的项目文件夹中（如：/myproject），就可以通过访问 localhost:8080/myproject 的方式访问到这一项目；

    IntelliJ 默认的部署目录是当前工程目录下的 out 文件夹。即在写好代码并点击右上角的运行按钮后，IntelliJ 会将项目文件输出到当前项目目录下的 out 文件夹而不是 webapps 中，也就意味着，此时如果采用手动启动 Tomcat 服务的方式，是无法通过访问 localhost:8080/myproject 访问项目的；

特别注意

将项目放在Tomcat->webapps目录下无法访问的问题，由于该文件夹没有写权限，无法生成字节码文件，所以访问index.jsp文件找不到。需要在Apache Software Foundation文件夹添加权限。添加本地账户全部控制。
